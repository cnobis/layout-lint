## 🎯 Project Summary

LayoutLint is a declarative DSL for testing CSS layout constraints in browsers. It uses Tree-sitter for parsing and evaluates spatial relationships between DOM elements using `getBoundingClientRect()`.

**Core Value Proposition:**
- Write `gallery below nav 5px` instead of brittle DOM queries
- Real-time validation during development
- Interactive demo with draggable elements
- 14 spatial relations covering 99% of layout testing needs

## 🔧 Build Workflow (IMPORTANT)

### Grammar Changes Require Special Steps

**Problem:** `tree-sitter generate` requires CommonJS, but project uses ESM (`"type": "module"`).

**Solution (from DEVELOPMENT.md):**
```bash
# 1. Remove "type": "module" from package.json
# 2. Run tree-sitter generate
tree-sitter generate
# 3. Restore "type": "module" to package.json
# 4. Rebuild everything
npm run build:ts && npm run build:wasm
```

### Normal Development (No Grammar Changes)
```bash
npm run build:ts      # Compile TypeScript
npm run serve         # Start demo server on :8080
npm test              # Run test suite
```

## 🎨 Current Features (v1.0.0)

### 14 Supported Relations

| Category | Relations | Example |
|----------|-----------|---------|
| **Directional** | `below`, `above`, `left_of`, `right_of` | `nav below header 10px` |
| **Alignment** | `aligned_top`, `aligned_bottom`, `aligned_left`, `aligned_right` | `logo aligned_left nav` |
| **Semantic** | `contains`, `overlaps` | `modal overlaps page` |
| **Size** | `wider_than`, `taller_than`, `same_width`, `same_height` | `sidebar wider_than 200px` |
| **Absolute** | `distance_from_top` | `header distance_from_top 0px` |

### Evaluation Logic (src/evaluator.ts)
- **Directional:** Pass if `actual >= threshold` (distance in px)
- **Alignment:** Pass if `actual <= 1px` (1px tolerance)
- **Semantic:** Pass if `actual === 0` (boolean: 0 = true, 1000 = false)
- **Size:** Pass if `actual >= 0` (positive difference)

## 🧪 Testing Strategy

**Current:** 8 tests in `test/basic.test.js` using Node.js native test runner.

**Coverage:**
- Directional relations (below, right_of)
- Semantic relations (overlaps)
- Alignment relations (aligned_left)
- Edge cases (missing elements, multiple rules)

**To Add Tests:**
1. Create `.test.js` file in `test/`
2. Import `evaluateRules` from `dist/evaluator.js`
3. Mock DOM elements with `getBoundingClientRect()`
4. Run `npm test`

**Example Test Pattern:**
```javascript
const rules = [{ element: 'nav', relation: 'below', target: 'header', distancePx: 10 }];
const mockResolve = (id) => ({
  getBoundingClientRect: () => ({ top: 110, bottom: 150, left: 0, right: 100 })
});
const results = evaluateRules(rules, mockResolve);
assert.strictEqual(results[0].pass, true);
```

## 🐛 Known Issues & Quirks

### 1. Grammar Build Process
**Issue:** Must remove/restore `"type": "module"` for `tree-sitter generate`  
**Impact:** Annoying but documented in DEVELOPMENT.md  
**Workaround:** Automate with script or accept manual steps

### 2. Hyphenated Element IDs
**Issue:** Originally didn't work, fixed in v1.0.0  
**Status:** ✅ Now supports `featured-badge`, `nav-bar`, etc.  
**Grammar:** `identifier: $ => /[a-zA-Z_][a-zA-Z0-9_-]*/`

### 3. Server Restart Issues
**Issue:** Occasional HTTP2 protocol errors during `npm run serve`  
**Workaround:** Kill server manually: `pkill -f http-server`, then restart  
**Not Critical:** Development only, doesn't affect package

### 4. Demo Image Paths
**Issue:** Initially used wrong relative paths  
**Fixed:** Now uses `pictures/art-gallery/{barbershop,bluewithred,swimmer}.jpeg`  
**Stable:** No further changes expected

## 💡 Development Best Practices

### When Adding a New Relation

1. **Update grammar.js:**
   ```javascript
   relation: $ => choice(
     "below", "above", "new_relation", // add here
   ```

2. **Follow grammar build workflow** (remove/restore `"type": "module"`)

3. **Add measurement logic to src/evaluator.ts:**
   ```typescript
   case "new_relation": return /* calculation */;
   ```

4. **Add test in test/basic.test.js:**
   ```javascript
   it('should handle new_relation', () => { /* ... */ });
   ```

5. **Update README.md** relations table

6. **Update CHANGELOG.md** with new feature

### When Fixing a Bug

1. **Add failing test** reproducing the bug
2. **Fix the code** until test passes
3. **Update CHANGELOG.md** under "Fixed"
4. **Bump patch version** (e.g., 1.0.0 → 1.0.1)

### When Updating Demo

1. **Edit demo/index.html** (HTML/CSS/JS all in one file)
2. **No build needed** (HTML is static)
3. **Refresh browser** (hard refresh: Cmd+Shift+R)
4. **Test interactive features** (drag, resize, constraints update)
